# Implementation Plan: Invoice PDF Generation and Preview

**Branch**: `007-invoice-pdf-preview` | **Date**: 2026-04-19 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/007-invoice-pdf-preview/spec.md`

## Summary

Extend the existing `SendInvoiceScreen` with full invoice detail fields (seller, buyer, line items, VAT, dates), submit to `POST /v1/invoices`, receive a ZUGFeRD PDF/A-3 binary, save it to a temp file, and render it in a native full-screen PDF viewer (`react-native-pdf`) without WebView. A share button lets users send or save the PDF via the native share sheet.

## Technical Context

**Language/Version**: TypeScript 5.x / React Native 0.76 (Expo SDK 52)  
**Primary Dependencies**: `react-native-pdf` (PDF viewer), `react-native-blob-util` (binary fetch + temp file), `expo-sharing` (share sheet) — all new; existing: Expo SDK 52, React Navigation v6, `useAuth()` for token  
**Storage**: Temp file only — `{CacheDir}/invoice-{invoiceNumber}.pdf` via `react-native-blob-util`; not persisted  
**Testing**: No test suite configured in project  
**Target Platform**: iOS 15.1+ (PDFKit), Android API 21+ (PdfRenderer)  
**Project Type**: Mobile app (Expo bare workflow)  
**Performance Goals**: PDF displayed within 5 seconds on standard mobile connection  
**Constraints**: No WebView; no server-side PDF persistence; logo upload out of scope  
**Scale/Scope**: Single-user, single invoice at a time; multi-page PDF support required
