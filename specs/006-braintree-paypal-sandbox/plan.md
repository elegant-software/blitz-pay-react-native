# Implementation Plan: Geofencing & Proximity Triggers

**Branch**: `006-braintree-paypal-sandbox` | **Date**: 2026-04-19 | **Spec**: `specs/006-braintree-paypal-sandbox/spec.md`
**Input**: Use geofencing (Android/iOS APIs) → triggers automatically when user enters merchant area; periodically fetch user location; proximity check locally or via backend API.

## Summary

Add native OS-level geofencing to the BlitzPay mobile app to detect when the user is physically at a registered merchant location. **This feature is location-only — it has no coupling to payments.** `expo-location` + `expo-task-manager` drive background geofence monitoring (power-efficient, no polling required). When a geofence `enter` event fires, the background task POSTs a proximity report to the backend API (which decides what to do with that signal) and optionally surfaces a neutral local notification. A configurable periodic polling mode is available as fallback for devices where background geofencing is restricted.

## Technical Context

**Language/Version**: TypeScript 5.3, React Native 0.83.4, Expo SDK 55  
**Primary Dependencies**: `expo-location` (^17.x), `expo-task-manager` (^12.x), `expo-notifications` (already installed)  
**Storage**: `expo-secure-store` (already installed) — persists active geofence region list across restarts  
**Testing**: Manual simulator testing (Xcode location simulation, Android emulator geo fixes); no automated test suite configured  
**Target Platform**: iOS 15.1+, Android (minSdk from Expo default ~24+)  
**Project Type**: Mobile app (React Native / Expo managed workflow with dev-client)  
**Performance Goals**: Geofence wake latency < 30s (native OS budget); polling interval ≥ 60s to preserve battery  
**Constraints**: Background location permission required (user-granted); iOS background modes must include `location`; Android `ACCESS_BACKGROUND_LOCATION` requires separate rationale dialog on API 29+  
**Scale/Scope**: Prototype — up to 50 merchant geofence regions; single user session
